# ROADMAP — 매매 지원 + 백테스팅 + 알림·ESG 시스템 확장 설계서

> **상태:** ✅ **Phase A + B + C + D + E + F (ESG·실시간데이터·카톡알림) 완료 (v6 구현 완료)**
> **목적:** 분석·추천·매매 의사결정·자가 평가·알림 통합 시스템
> **현재 버전:** v6 (26 에이전트, 10-Phase + 백테스팅 + 알림, 7 Modes A~G)

---

## 1. 설계 철학

본 로드맵은 다음 경계를 유지한다:

| 다룬다 | 다루지 않는다 |
|---|---|
| 분석 → 추천 → 매매 의사결정 보조 | 자동 매매 실행 |
| 정량적 포지션 사이징 | 증권사 API 직접 연동을 통한 주문 |
| 보유 종목 모니터링·재평가 | 알고리즘 트레이딩 |
| 세금·비용 인식 | 세무 신고 자동화 |

**핵심 원칙:** AI는 정보·분석·시뮬레이션을 제공하고, **최종 매매 결정과 실행은 사용자 본인**이 한다. 이 경계를 넘으면 법적·규제 리스크와 책임 소재 문제가 발생한다.

## 2. 현재 시스템 (v2) 한계 진단

매매 관점에서 현재 시스템의 부족함:

| 영역 | 현재 v2 수준 | 부족한 점 |
|---|---|---|
| 진입 타이밍 | "1차/2차/3차 분할 매수 가격" 정적 제시 | 실시간 시장·변동성·뉴스 반영 X |
| 익절·손절 | 고정 가격대 | 변동성·시장 국면 기반 동적 조정 X |
| 포지션 사이징 | "보수 3~5%" 정성 권고 | Kelly Criterion·ATR 기반 정량 계산 X |
| 포트폴리오 관점 | 단일 종목만 분석 | 보유 종목 상관관계·섹터 집중도 모름 |
| 시장 환경 | 종목 내부 분석만 | 거시 환경·VIX·금리 컨텍스트 미반영 |
| 매매 이후 추적 | 추천 기록 (JSONL) 저장만 | 진입 후 모니터링·자동 재평가 트리거 없음 |
| 세금·비용 | 무시 | 거래세·양도세·수수료 미반영 |
| 자기 학습 | 추천 기록은 있으나 | 백테스팅·정확도 측정 자동화 없음 |

## 3. 신규 추가 후보 기능 (12개)

### Tier 1 — 매매 의사결정에 결정적 (★ 최우선)

#### 3.1 `portfolio-analyst` — 포트폴리오 컨텍스트

**Why:** 같은 BUY 추천이라도 보유 종목 구성에 따라 매수 적정성이 달라진다. 반도체 비중이 이미 40%인데 메모리 종목 추가는 분산 위반.

**기능:**
- 사용자 입력 또는 `portfolio.json` 로드
- 신규 분석 종목 vs 기존 보유 **상관관계 분석**
- **섹터·시가총액·통화·지역 집중도** 점검
- 신규 매수 시 **포트폴리오 전체 베타** 변화 추정
- 권고: "현재 포트폴리오 적합 / 분산 위반 / 보완 효과"

**입력 형식:**
```json
{
  "holdings": [
    {"ticker": "005930", "name": "삼성전자", "weight": 25, "avg_price": 72000},
    {"ticker": "AAPL", "name": "Apple", "weight": 15, "avg_price": 180}
  ],
  "cash_ratio": 30,
  "total_value_krw": 100000000
}
```

**의존성:** 독립 실행 가능. advisor 호출 전후 모두 가능.

#### 3.2 `market-regime-analyst` — 시장 환경

**Why:** 강세장 BUY와 약세장 BUY는 다르게 다뤄야 한다. 같은 종목 같은 점수라도 시장 국면이 다르면 진입 전략이 달라진다.

**기능:**
- **현재 시장 국면 판단**: Bull / Bear / Sideways / High Volatility
- **추적 지표:**
  - VIX, V-KOSPI (변동성 지수)
  - 미국 10년물 국채 금리·금리곡선 (역전 여부)
  - DXY (달러 인덱스)
  - 한국 외국인 누적 순매수
  - 코스피·S&P500의 200일선 위치
  - Fear & Greed Index
- **국면별 advisor 가중치 조정 권고:**
  - 약세장 → 손절선 타이트화, 포지션 50% 축소
  - 고변동성 → 분할 매수 단계 증가 (3 → 5단계)
  - 강세장 → 트레일링 스탑 활용 강화

**의존성:** advisor 단계 전에 실행해 advisor에 컨텍스트로 전달.

#### 3.3 `position-sizer` — 정량 포지션 사이징

**Why:** "5%로 매수하세요"가 아닌 수학적 근거 기반 사이징.

**기능:**

**(a) Kelly Criterion 적용:**
```
f* = (p × b - q) / b
  p = Bull/Base 실현 확률 (advisor 시나리오 활용)
  b = 평균 수익률 / 평균 손실률
  q = 1 - p
```
Full Kelly는 리스크 과대 → **Half-Kelly 또는 Quarter-Kelly** 권고.

**(b) ATR(Average True Range) 기반 손절폭:**
```
손절폭 = 2 × ATR(14)
포지션 크기 = (총 자산 × 리스크 허용비율) / 손절폭
```
예: 총 자산 1억 × 리스크 허용 1% = 100만원 손실 한도 / 종목 ATR 기반 손절폭 = 매수 가능 주수.

**(c) 변동성 타게팅:**
종목 σ 기반 비중 조정 (변동성 높을수록 비중 ↓).

**의존성:** advisor의 시나리오 확률·목표가·손절선 필요. advisor 후 실행.

#### 3.4 `entry-timing-checker` — 진입 타이밍

**Why:** "오늘 사도 되는가, 며칠 더 기다리는가?" 추천이 BUY라도 단기 과열 상태면 며칠 지연 가치 있음.

**기능:**

**단기 과열 점검:**
- RSI(14) > 70 → 과매수 경고
- 볼린저밴드 상단 이탈
- 5/10/20일선 대비 이격도

**임박 이벤트 점검:**
- 다음 분기 실적 발표까지 D-X일
- FOMC·한은 금통위까지 D-X일
- 옵션 만기일까지 D-X일

**출력:**
- "즉시 1차 매수 가능"
- "이벤트 통과 후 매수 (D+X)"
- "단기 조정 대기 (RSI X로 하락 시 진입)"

**의존성:** advisor의 진입 가격대 필요. advisor 후 실행.

### Tier 2 — 매매 후 추적·재평가 (지속 가치)

#### 3.5 `position-monitor` — 보유 종목 모니터링

**Why:** 매수 후 방치하면 추천 무효화 시점을 놓침. v2의 `invalidation_triggers`를 실제로 추적해야 가치 발휘.

**기능:**
- `reports/[티커]_recommendations.jsonl`의 기록 자동 점검
- 다음 조건 만족 시 알람:
  - 손절선 도달
  - 무효화 트리거 발생 (예: "분기 매출 YoY -10% 이하")
  - 목표가 도달 → 익절 검토
  - 추천 후 X일 경과 시 자동 재평가
- 출력: 액션 리스트 (HOLD / REDUCE / EXIT / RE-ANALYZE)

**의존성:** v2의 JSONL 기록 시스템 (이미 존재). 추가 의존성 없음.

#### 3.6 `reanalysis-trigger` — 자동 재분석 트리거

**Why:** 시장 상황 변화 시 직전 추천이 유효한지 자동 점검.

**기능:**
- 직전 추천 후 다음 이벤트 발생 시 자동 트리거:
  - 분기 실적 발표 (전후 ±3일)
  - 가이던스 변경 공시
  - 주가 ±15% 변동
  - 동종 산업 주요 사건 (경쟁사 실적, 규제 변경)
- 재분석 시 직전 추천 결과를 컨텍스트로 활용 → "유지/조정/철회" 권고
- 오케스트레이터의 Phase 0 강화 형태

**의존성:** v2의 JSONL 기록 + 시장 데이터 (MCP 권장).

#### 3.7 `portfolio-rebalancer` — 리밸런서

**Why:** 시간 경과 시 비중 어긋남. 정기 리밸런싱이 장기 수익률에 영향.

**기능:**
- 현재 비중 vs 목표 비중 비교
- **세제 효율적 리밸런싱** 권고:
  - 매도 시 양도세 계산
  - 손실 종목 우선 매도 (Tax-loss harvesting)
- 분기·반기 단위 자동 점검

**의존성:** `portfolio-analyst` + `tax-cost-calculator` 결합 시 효과 극대화.

### Tier 3 — 실전 매매 정밀도

#### 3.8 `tax-cost-calculator` — 세금·비용 계산

**Why:** 명목 수익률과 실수익률은 다름. 매매 결정 전 세후 수익률 인식 필수.

**기능:**

**한국 주식:**
- 거래수수료 (증권사별)
- 거래세 (코스피 0.18%, 코스닥 0.18% — 매도 시)
- 대주주 양도세 (해당 시 22~33%)
- 배당소득세 15.4%

**미국 주식:**
- 양도소득세 22% (250만원 공제 후)
- 환차익 영향
- 배당세 (이중과세, 14% 또는 15%)

**출력:** 명목 수익률 vs 세후 실수익률, 손익분기 매도 가격.

**의존성:** 독립 실행. advisor 결과에 후처리로 적용.

#### 3.9 `order-book-analyst` — 호가창 분석

**Why:** 분할 매수 1차 시점에 호가창 상태가 진입 가격 결정에 영향.

**기능:**
- 매수·매도 호가 잔량 비율
- 최근 5분 체결강도
- 대량 매도호가 (매도벽) 위치
- 권고: "지정가 vs 시장가", "분할 시간 간격"

**의존성:** ⚠️ **실시간 시세 API 필요** (한국투자증권 KIS, 키움 OpenAPI+ 등). MCP 통합 가능 시점에 도입.

#### 3.10 `hedge-advisor` — 헷지 검토

**Why:** 큰 포지션은 다운사이드 헷지 가치. 단순 매수자에겐 불필요.

**기능:**
- Protective Put (개별 옵션) 매수 비용 vs 효과
- 코스피200 풋옵션 (전체 헷지)
- 인버스 ETF 활용
- 헷지 비용이 기대 수익의 X% 이내일 때 권고

**의존성:** 옵션 가격 데이터. MCP 통합 권장.

#### 3.11 `fx-hedge-analyst` — 환헷지 분석 (미국 주식 전용)

**Why:** 미국 주식 매수 시 환율 노출도 점검. 강달러 vs 약달러 시나리오에서 실현 수익률 차이.

**기능:**
- 환율 노출도 분석
- 환헷지 ETF (예: KODEX 미국S&P500선물(H)) vs 직접 매수 비교
- 환율 시나리오별 실현 수익률
- 환헷지 비용 vs 기대 환위험 손실

**의존성:** 환율 시계열 데이터.

### Tier 4 — 자동화·통합

#### 3.12 `daily-briefing` — 일일 브리핑

**Why:** 매일 정기 점검으로 시장·종목 변화 추적.

**기능:**
- 매일 아침 자동 생성:
  - 어제 주요 시장 이벤트
  - 보유 종목 관련 뉴스·공시
  - 오늘 발표 예정 일정 (실적, 경제지표)
  - 모니터링 중인 트리거 상태
- Cron job 또는 수동 호출

**의존성:** `portfolio-analyst` + `position-monitor` + 뉴스 데이터.

## 5. 단계별 구현 로드맵

### ✅ Phase A — 매매 컨텍스트 확장 (완료)

**목표:** advisor의 추천을 "단일 종목 분석" → "포트폴리오·시장 환경 반영 매매 권고"로 격상.

**추가 에이전트 (구현 완료):**
- `portfolio-analyst` (blue)
- `market-regime-analyst` (blue)
- `position-sizer` (green)
- `entry-timing-checker` (red)

**오케스트레이터 변경:** 6-Phase → 10-Phase 워크플로우.

### ✅ Phase B — 추적·재평가 자동화 (완료)

**목표:** 매수 후 방치 방지. 시간 경과 시 추천 유효성 자동 점검.

**추가 에이전트 (구현 완료):**
- `position-monitor` (orange)
- `reanalysis-trigger` (orange)
- `portfolio-rebalancer` (orange)

### ✅ Phase C — 정밀도 향상 (완료)

**추가 에이전트 (구현 완료):**
- `tax-cost-calculator` (green)
- `hedge-advisor` (green)
- `fx-hedge-analyst` (green)

### ✅ Phase D — 고급 기능 (완료, 일부 제약)

**추가 에이전트 (구현 완료):**
- `order-book-analyst` (red) — ⚠️ 실시간 시세 MCP 미통합 시 구조적 권고만
- `daily-briefing` (blue)

**남은 제약:** 실시간 시세·옵션·환율 데이터 MCP 통합 시 정밀도 향상 가능.

### ✅ Phase E — 백테스팅·시스템 자가 평가 (완료, v5 신규)

**목표:** 시스템 자체의 성과를 측정하고, 발견된 약점을 기반으로 자가 개선.

**추가 에이전트 (구현 완료):**
- `backtest-collector` (yellow) — reports/*.jsonl + 실제 가격 매칭 → backtest_results.jsonl
- `backtest-analyzer` (yellow) — 6개 핵심 지표 (Win Rate, Profit Factor, Sharpe, 의견별 정확도, devils-advocate 가치, 시나리오 보정) + 세그먼트 분석 + 통계적 함정 점검
- `backtest-optimizer` (yellow) — 발견된 약점 → 구체적 시스템 개선 권고 (과적합 방지 검증 포함)

**오케스트레이터 변경:** Mode F (백테스팅 사이클) 추가.

**제약:** 첫 의미 있는 결과는 추천 누적 30+ + 12개월 경과 후 (사용 시작 약 1년 후).

### ✅ Phase F — ESG·실시간 데이터·외부 알림 (완료, v6 신규)

**목표:** 종목 분석의 비재무 차원(ESG) 통합, 실시간 데이터 접근 표준화, 외부 알림으로 사용자가 시장 변동을 놓치지 않게.

**추가 에이전트 (구현 완료):**
- `esg-analyst` (red) — 환경(10) + 사회(10) + 거버넌스(5) = 25점, risk-analyst와 통합. Phase 2 분석가가 5명 → 6명.
- `realtime-data-fetcher` (blue) — 시스템 인프라. Alpha Vantage/Polygon/KIS/Yahoo MCP 자동 라우팅 + 표준 스키마. 다른 에이전트들이 일관되게 데이터 소비.
- `kakao-notifier` (orange) — PlayMCP `KakaotalkChat-MemoChat` 통합. 200자 자동 분할 + 우선순위 + 24시간 중복 방지 + Quiet Hours.

**오케스트레이터 변경:**
- Phase 2를 6명 분석가로 확장 (ESG 추가)
- advisor가 ESG 점수를 risk_inverse Pillar에 통합
- Mode G (카톡 알림) 추가 — 자동 트리거 + 수동 요청
- 자동 트리거 조건: 손절선·무효화·목표가·신규 Strong BUY·시장 국면 전환·백테스팅 약점 발견 시

**제약:**
- 카톡 알림은 PlayMCP 연결 필수
- PlayMCP MemoChat은 메시지당 200자 한도 (분할 처리 필수)
- 실시간 시세 MCP가 없으면 realtime-data-fetcher는 WebSearch 폴백
- ESG 평가는 기관별 점수 차이가 큼 (다중 출처 교차 검증 필수)

## 8. 리스크 및 제약

### 8.1 시스템 한계

- LLM은 미래 시장을 예측할 수 없다. 정확도 향상은 보조적이다.
- 정량 모델 (Kelly, ATR)도 과거 데이터 기반이며 미래 보장 없음.
- 환각·잘못된 데이터 해석 가능성 상존 → 사용자 검증 필수.

### 8.2 법적·규제

- **자동 매매 실행 기능은 본 로드맵에 포함하지 않는다.** 한국 자본시장법상 투자일임업 등록 등 문제 발생 가능.
- 본 시스템은 정보 제공·분석 보조 도구로만 활용.

### 8.3 책임

- 모든 매매 결정은 사용자 본인 책임.
- 시스템 권고는 참고 자료이며, 매매 실행 전 사용자가 검증해야 함.

backtest-collector / backtest-analyzer / backtest-optimizer 3개 에이전트로 완성.

### 11.3 ✅ ESG·지속가능성 분석 (v6에서 완료)

esg-analyst 에이전트로 완성. risk-analyst와 통합.

`memory` 필드 + backtest-optimizer 자동화:
- 백테스팅 결과를 메모리에 누적
- 에이전트별 약점 자동 파악

### 11.8 의사결정 보조 vs 실행의 경계 유지

v7+에서도 다음 경계는 **유지**한다:
- ❌ 자동 매매 실행
- ❌ 세무 신고 자동화
- ❌ 사용자 인증·계좌 접근

이 경계를 넘는 순간 법적·규제 리스크와 책임 소재 문제가 발생하며, AI 보조 도구의 본질을 벗어난다.
